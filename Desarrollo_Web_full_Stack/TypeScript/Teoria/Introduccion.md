## TypeScript.

Entonces TypeScript se creo a partir de una necesidad la cucal era hacer que el lenguage de javascript fuera un lenguage cocmpleto de programacicon y su completa reutilizacion de condigo en proyectos extensos y le hacen falta muchas mas cosas como:

* El tipado de variables.
* Errores en el tiempo de escritura.
* El auto completado dependiendo de variables.
* Clases y modulos: su uso tiene algunos inconvenientes ya que algunos no son soportados en cuanto a sus propiedades y metodos privados.
* Validacion de objetos dentro de objetos es muy escasa.
* No tenemos las formas para mapear peticiones http, ya que no hay una manera especifica o concreta para saber que lugares se deben cambiar en la applicacio nsi modificamos algo de lo creado con anterioridad.

Entonces teniendo en cuenta estos problemas, lo que busca TypeScript es tener una experiencia similar al desarrollar como en otros lenguages de programacion como Java, C#, Objective C.

Es como un Super set de JavaScript pero bno es capaz de correr de forma directa en el navegador web, lo que hace es que "transpila" el codigo a JavaScript , pero no solo hace esto sino que tambien añade muchas mas funcionalidades y mejoras y pasarlas a el estandar de Js que se necesita.

    Diferencia entre transpila y compilar: compilar es pasar el codigo a lenguage de maquina osea binario haciendo que se ejecute y transpilar es traducir un codigo de una forma a otra pero se hace lo mismo.

Permitiendo asi escribir un codigo moderno el cual será soportado por casi todas las plataformas y navegadores.

### Instalaciones:

* Debemos instalar node para hacer pruebas e instalar diferentes cosas.
* Descargar un editor de codigo para con apoyo para TypeScript
* Instalar extencion Activitus Bar, que nos sirve para tener una barra extra en nuestra parte inferior.
* Luego instalar TypeScrript Importer para realizar diferentes autoimportaciones de forma rapida en TSI.
* Instalar esta extension para google Chrome JSON Viewer para visualizar los archivos .json de manera mas efectiva.

Al Trabajar con TypeScript, ayuda mucho a cometer menos errors de programación por el costo de más código y tiempo de desarrollo, pero lo recuperamos a la hora de refactorizar o encontrar errores en nuestro programa a la hora de escribirlo.
